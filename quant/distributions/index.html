Every outcome has a shape.

A probability distribution is just a complete map of every possible outcome and how likely each one is. This module builds that map for the five distributions investment analysts reach for constantly — starting with a coin flip and ending with the curves behind option pricing, portfolio risk, and statistical testing.

01

Random Variables & Probability Distributions

Two views of the same information: "what's the chance of exactly this?" and "what's the chance of this or less?"

A random variable is discrete if you can count its possible outcomes — even if the list goes on forever — and continuous if you can't, because between any two possible values there's always another one. The number of "yes" votes at a board meeting is discrete; a stock's daily return is continuous.

Two views of a distribution

Probability function — the chance of one exact outcome. For discrete variables, p(x) = P(X = x), the probability mass function. For continuous variables, f(x), the probability density function (pdf) — but note a continuous variable's probability of hitting any single exact value is always 0; only ranges have positive probability.

Cumulative distribution function (cdf) — the chance of that outcome or less: F(x) = P(X ≤ x). It accumulates the probability function up through x.

The two properties every probability function must satisfy
0 ≤ p(x) ≤ 1   and   Σ p(x) over all x = 1
Worked example — rolling a die

p(3) = 1/6 — the pdf value at 3. F(3) = P(X ≤ 3) = p(1)+p(2)+p(3) = 3/6 = 50% — the cdf at 3. Note the jump: F only changes at the six possible outcomes, since nothing lies between them for a discrete variable.

Worked example — EAA Index daily returns (continuous)

Binning five years of daily returns: 0.1% of days fell in the −5% to −4% bin, 0.6% in the −4% to −3% bin, and 1.8% in the −3% to −2% bin. The cdf at −2% sums these: F(−2%) = 0.1% + 0.6% + 1.8% = 2.5% — the probability of a return below −2%, treating the historical pdf as a guide for the future.

Correct — price can take on essentially any value in a range (continuous), while a rating is one of a finite, countable list of categories (discrete, technically ordinal categorical here).

The Discrete Uniform Distribution

The simplest distribution of all: every outcome is equally likely.

With n equally likely outcomes, each one simply gets probability 1/n. That's the entire distribution — nothing more to specify.

Discrete uniform pdf
p(x) = 1/n  for each of the n possible outcomes

Reading a cdf table lets you answer range questions two equivalent ways — by summing individual probabilities, or by subtracting two cumulative values:

Worked example — outcomes 1 through 8

With n = 8, p(x) = 1/8 = 0.125 for every outcome. What's P(4 ≤ X ≤ 6)?

Direct sum: p(4)+p(5)+p(6) = 0.375. Or via the cdf: F(6) − F(3) = 0.750 − 0.375 = 0.375 — same answer either way. Watch the boundary carefully: P(4 < X ≤ 6) instead uses F(6) − F(4) = 0.250, since it excludes 4 itself.

Check yourself
A discrete uniform random variable has 10 equally likely outcomes: 1 through 10. What is P(X ≤ 4)?
Correct — F(4) = 4 × (1/10) = 0.40, since 4 of the 10 equally likely outcomes are ≤ 4.
Try it — discrete uniform explorer
P(4 ≤ X ≤ 6) = 0.375
F(6) − F(3) = 0.750 − 0.375 = 0.375
03

The Continuous Uniform Distribution

Same idea, stretched across a continuous range — a flat line instead of equal spikes.

Between limits a and b, the density is constant — a flat horizontal line — because every value in the range is equally likely. Probability over any sub-interval is simply that interval's share of the total width.

Continuous uniform pdf & cdf
f(x) = 1/(b − a)  for a ≤ x ≤ b, else 0
F(x) = (x − a)/(b − a)  for a ≤ x ≤ b
Mean & variance
μ = (a + b)/2     σ² = (b − a)² / 12
Worked example — a loan covenant breach

Interest charges are a fixed $25M; EBITDA is equally likely to fall anywhere from $40M to $60M. The interest coverage ratio (EBITDA/interest) is therefore continuous uniform between a = 1.6 and b = 2.4. What's the probability the ratio falls below the 2.0 covenant trigger?

F(2.0) = (2.0 − 1.6)/(2.4 − 1.6) = 0.4/0.8 = 50%. Mean = (1.6+2.4)/2 = 2.0; variance = (0.8)²/12 = 0.0533, so σ ≈ 0.231 — though note standard deviation is a less natural risk measure here than for a bell-shaped distribution, since a uniform distribution has no "typical" clustering near the mean.

Check yourself
X is continuous uniform between 10 and 30. What is P(X ≤ 15)?
Correct — F(15) = (15−10)/(30−10) = 5/20 = 0.25.

The Binomial Distribution

The distribution behind every "success or failure, how many times?" question.

Start with the simplest possible random variable: a Bernoulli trial, which succeeds with probability p and fails with probability 1−p. Flip that same coin n times, independently, and count the successes — that count is a binomial random variable, written X ~ B(n, p).

The two assumptions behind the binomial model

1. The probability of success, p, is constant across all trials.

2. The trials are independent of each other.

To land on exactly x successes in n trials, any specific sequence with x successes has probability px(1−p)n−x. Multiply by the number of different sequences that produce x successes — the combination formula from counting — and you get the full binomial probability function:

Binomial probability function
p(x) = P(X = x) = [n! / ((n−x)! x!)] · px (1−p)n−x
Worked example — a 4-period binomial stock tree

A stock moves up or down each of 4 independent periods, p = 0.50 either way. How many distinct paths produce exactly 3 up-moves? ₄C₃ = 4!/(1!3!) = 4 — the sequences uud u, uudu, uduu, duuu (each with probability 0.5⁴ = 0.0625).

p(3) = 4 × 0.0625 = 25%. The distribution is perfectly symmetric here because p = 0.50 — it becomes skewed whenever p departs from 0.50.

A binomial tree for n = 4 periods. Each node splits into an up-move and a down-move; the number of distinct paths reaching each ending node is exactly the combination formula ₄Cₓ.

Check yourself
In a binomial model with n = 5 and p = 0.5, how many distinct sequences produce exactly 2 successes?
Correct — ₅C₂ = 5!/(3!2!) = 10.

05

Binomial Mean, Variance & Real Applications

Two clean formulas, and two everyday uses: auditing trading desks and forecasting bond defaults.

Mean & variance of a binomial random variable
E(X) = np     Var(X) = np(1 − p)

A Bernoulli variable is just the special case n = 1: mean p, variance p(1−p).

Worked example — expected bond defaults

A high-yield portfolio holds 25 bonds, each with an independent 10.7% annual default probability. Expected defaults: E(X) = np = 25 × 0.107 = 2.675, roughly 3 bonds.

Variance = np(1−p) = 25 × 0.107 × 0.893 = 2.389, so σ ≈ 1.55. A rough "2-sigma" band around the mean runs from about 0 to 6 defaults — though in reality, bond defaults tend to cluster together during downturns, violating the independence assumption and understating true tail risk.

Worked example — auditing a block broker

A broker's trades were profitable 5 times and unprofitable 3 times out of 8 total trades last month. If the broker were truly giving you fair-priced trades on average (p = 0.50), how likely is a result this good or better (5 or more profitable trades) just by chance?

P(X ≥ 5) = p(5)+p(6)+p(7)+p(8) = 0.219+0.109+0.031+0.004 = 36.3%. That's a substantial chance of seeing this result even from a perfectly fair broker — not strong evidence you're getting better-than-fair pricing, just yet.

Check yourself
A binomial random variable has n = 10 and p = 0.30. What are its mean and variance?
Correct — E(X) = np = 10(0.3) = 3.0. Var(X) = np(1−p) = 10(0.3)(0.7) = 2.1.

Properties of the Normal Distribution

The bell curve — the single most-used distribution in investment work.

The normal distribution's importance traces back to the central limit theorem: the sum or mean of many independent random variables tends toward a normal distribution, regardless of the shape of the original variables. That single fact is why the bell curve shows up everywhere in statistics.

Defining characteristics

Two parameters, completely describe it: mean μ and variance σ². Written X ~ N(μ, σ²).

Symmetric: skewness = 0. Mean = median = mode.

Kurtosis = 3 (excess kurtosis = 0) — the reference point for "normal" tail thickness.

Closed under linear combination: a weighted sum of normal random variables is itself normally distributed — which is exactly why a portfolio of normally-distributed stock returns is itself normally distributed.

A single normal variable has a univariate distribution. A group of related variables — say, returns on 30 stocks — has a multivariate distribution, which additionally requires knowing every pairwise correlation between them.

Parameters needed for a multivariate normal distribution of n assets
n means  +  n variances  +  n(n−1)/2 distinct pairwise correlations

A real caveat: actual equity returns usually show kurtosis above 3 — "fat tails" — meaning extreme moves happen more often than the normal distribution predicts. The normal model also fits quarterly and annual returns better than daily or weekly returns, and it isn't suitable for asset prices at all (which can't go below zero) — that's what the lognormal distribution is for.

Check yourself
A portfolio's return is a weighted average of 20 individual stock returns, each of which is normally distributed (jointly). What can you say about the portfolio's return distribution?
Correct — a linear combination (weighted average) of jointly normal random variables is always normally distributed, regardless of the correlations between them.

Two normal distributions, same mean (μ=0), different standard deviations. The tighter curve (σ=1) concentrates far more probability near the mean than the wider one (σ=2).

07

Probabilities Using the Normal Distribution — the Empirical Rule

Four numbers worth memorizing — they answer most "how likely is this outcome" questions at a glance.

The empirical rule
~50% of observations fall within μ ± (2/3)σ
~68% of observations fall within μ ± 1σ
~95% of observations fall within μ ± 2σ  (precisely, ±1.96σ)
~99% of observations fall within μ ± 3σ  (precisely, ±2.58σ)
Worked example — a portfolio's return range

A portfolio has mean return 12% and standard deviation 22%, assumed normal. About 95% of yearly outcomes should fall between 12% − 2(22%) = −32% and 12% + 2(22%) = 56% — a genuinely wide range, reflecting real equity volatility.

Check yourself
A fund has a normally distributed return with mean 8% and standard deviation 15%. Approximately what interval captures 68% of outcomes?
Correct — the ±1σ interval is 8% ± 15% = −7% to 23%, capturing about 68% of outcomes.

Nearly all of a normal distribution's probability (99.7%) sits within 3 standard deviations of the mean — which is exactly why standard deviation is such a useful, compact way to describe risk for anything roughly bell-shaped.

Standardizing a Random Variable & the Standard Normal

Translate any normal variable into a universal scale, and one table (or one calculator) answers every probability question.

Every normal distribution is different — different mean, different spread — which makes them hard to compare directly or look up in a single table. The fix: convert any X ~ N(μ, σ²) into Z, the number of standard deviations X sits from its own mean. Z always follows the standard normal distribution, N(0,1), no matter what X's original mean and variance were.

Standardization (the Z-score)
Z = (X − μ) / σ
Worked example

X ~ N(5, 1.5²). What's Z for X = 9.5? Z = (9.5 − 5)/1.5 = 3.00. The probability of observing X ≤ 9.5 is identical to the probability of observing Z ≤ 3.00 on the standard normal — that shared number is what makes standardization so useful.

A handful of standard-normal percentile points come up constantly enough to be worth memorizing:

Worked example — a portfolio VaR-style question

Portfolio return ~ N(12%, 22%²). What's the probability return exceeds 20%?

Z = (20% − 12%)/22% = 0.364. P(Z > 0.364) = 1 − P(Z ≤ 0.364) = 1 − 0.6419 = 35.8%.

Check yourself
X ~ N(50, 10²). What is the Z-score for X = 65?
Correct — Z = (65 − 50)/10 = 1.50.

09

Student's t-Distribution

The normal distribution's more cautious cousin — used whenever you're estimating, not just assuming, the standard deviation.

The t-distribution is symmetric and bell-shaped, just like the normal — but with fatter tails, meaning it assigns more probability to extreme outcomes. It's defined by a single parameter: degrees of freedom (df).

Where degrees of freedom comes from

The sample variance formula, s² = Σ(Xᵢ − X̄)²/(n−1), divides by n−1 because once the sample mean is fixed, only n−1 of the n observations are free to vary — the last one is determined by the requirement that they average to X̄. That n−1 is the degrees of freedom.

While Z = (X̄ − μ)/(σ/√n) is exactly standard normal, swap in the estimated standard deviation s instead of the true σ, and the ratio t = (X̄ − μ)/(s/√n) follows a t-distribution with n−1 degrees of freedom instead — the extra uncertainty from estimating s (rather than knowing it) is exactly what fattens the tails.

Drag the degrees of freedom down and watch the tails fatten relative to the standard normal (dashed). As df grows, the t-distribution converges to the normal.

Check yourself
As the degrees of freedom of a t-distribution increase, what happens to its shape?
Correct — as sample size (and therefore df) grows, the t-distribution's tails thin out and it approaches the standard normal shape.

Why it matters in practice: in real statistical work, you almost never know the true population standard deviation — you estimate it from the sample. That's precisely when the t-distribution, not the normal, is the technically correct tool, especially with smaller samples.

Chi-Square & F-Distributions

Two more df-driven distributions, both bounded below by zero and both essential for hypothesis testing.

The chi-square distribution

The distribution of a sum of k squared independent standard normal random variables. Because it's built from squared terms, it can never be negative — and unlike the normal or t, it's asymmetric, with a long right tail. Its single parameter is degrees of freedom, k. As k grows, the shape gradually becomes more bell-like.

The F-distribution

Also asymmetric and bounded below by zero, the F-distribution is what you get from the ratio of two chi-square variables, each divided by its own degrees of freedom: F = (χ₁²/m) / (χ₂²/n). It therefore needs two degrees-of-freedom parameters — numerator (m) and denominator (n) — and, like chi-square, becomes more bell-shaped as both grow large.

Where these show up

Both distributions are workhorses of statistical inference rather than models for asset returns directly: chi-square underlies tests about a single variance, and the F-distribution underlies tests comparing two variances or testing overall significance in regression — topics covered in depth in the sampling, estimation, and hypothesis testing modules that follow this one.

Which of these distributions can take on negative values?
Correct — both are built from squared (chi-square) or ratio-of-squared (F) terms, so neither can ever be negative.
